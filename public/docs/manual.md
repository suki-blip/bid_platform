## 1. מבוא והתחלת עבודה

### מה המערכת עושה
המערכת היא CRM לעמותות גיוס. היא מאפשרת:
- ניהול בסיס נתונים של תורמים ולידים
- רישום הבטחות תרומה (פלאגים) וחיובים
- חיוב אוטומטי בכרטיס אשראי דרך Sola/Cardknox
- שליחת קבלות אוטומטיות ודיוורים
- מעקב אחר גבייה (חיובים שעברו תאריך)
- דוחות ותקציבים
- שיתוף פעולה צוותי (מנהל + fundraisers)

### תהליך התחלת עבודה (Setup ראשוני)
**למנהל בלבד**, פעם אחת בלבד:

1. **חיבור שער התשלומים** — `תפריט משתמש → Settings → Sola/Cardknox`
   - הזן את ה-PaymentSITE URL שלך מ-Cardknox
   - הזן xKey (transaction key)
   - הזן ifields key
   - שמור

2. **חיבור שליחת אימיילים** — `Settings → Email`
   - From: `"שם הארגון <office@yourdomain.org>"`
   - הזן Resend API key (התחבר ל-resend.com → צור API key חדש)
   - הוסף signature HTML שיתווסף לכל אימייל
   - שמור
   - **חשוב**: שלח לעצמך אימייל בדיקה כדי לוודא שהכל עובד

3. **אימות דומיין ב-Resend** (אם רוצים לשלוח מ-`@yourdomain.org`)
   - ב-Resend dashboard → Domains → Add domain
   - הוסף את 3 הרשומות (SPF, DKIM, MX) ב-DNS של הדומיין שלך
   - המתן 5-30 דקות עד שהאימות יושלם

4. **הוספת תורם ראשון** — `Donors → New donor`
   - מלא שם פרטי, שם משפחה, שם עברי, אימייל
   - שמור

5. **יצירת קמפיין ראשון** — `Campaigns → + New campaign`
   - תן שם (לדוגמה "Annual Drive 2026")
   - הגדר יעד (Goal amount) אופציונלי
   - שמור

מעכשיו אתה מוכן לעבוד.

## 3. תורמים ולידים

### מה ההבדל?
- **Lead (ליד)** — מישהו שעדיין לא תרם. אנחנו עוקבים אחריו, מבצעים שיחות, מעבירים בין fundraisers.
- **Donor (תורם)** — מי שכבר תרם לפחות פעם אחת. אוטומטית עובר מ-Lead ל-Donor ביום התרומה הראשונה.

### הוספת תורם / ליד חדש
1. סרגל עליון → **Donors** (או **Leads**)
2. כפתור כהה בפינה הימנית: **+ New donor** / **+ New lead**
3. מלא:
   - **First name** (חובה)
   - **Last name** (אופציונלי)
   - **Hebrew name** — שם מלא בעברית
   - **Hebrew first name** + **Hebrew last name** — לחילופין שם פרטי + משפחה בעברית בנפרד
   - אימייל, טלפון, ארגון, מקור, תאריכים (יום הולדת, יארצייט, יום נישואין)
4. סמן **Do not contact** אם התורם ביקש שלא לפנות אליו
5. שמור — תועבר לפרופיל התורם

### עריכת פרטי תורם
1. **Donors** → לחץ על שורת תורם → נפתח פאנל צד
2. בפאנל: **Edit details** (או לחיצה על השם להיכנס לפרופיל המלא)
3. ערוך כל שדה
4. **Save** או **Esc** לסגירה ללא שמירה

### חיפוש וסינון תורמים
**שורת הסינון** מעל הרשימה:
- **תיבת חיפוש** — שם פרטי / משפחה / שם עברי / אימייל / ארגון
- **Source** — סנן לפי מקור (Newsletter, Referral, וכו')
- **Chip "Lapsed (12m+)"** — רק תורמים שלא תרמו 12 חודש
- **Chip "Has open pledges"** — רק תורמים עם פלאג פתוח (יש להם outstanding)
- **כפתור Sorted by X ▼** — מופיע כשמיון פעיל; לחץ לבטל

**מיון לפי עמודה:**
- לחץ על כותרת עמודה כדי למיין
- לחיצה שנייה — כיוון הפוך
- לחיצה שלישית — מבטל מיון
- העמודות שניתן למיין: Name, Rating, Contact, Organization, Source, Pledged, Paid, Last contact, Assigned

### פעולות גורפות על תורמים
1. סמן 2 או יותר תורמים ב-checkbox (העמודה השמאלית)
2. **סרגל כחול נפתח** בראש הטבלה: `"3 selected"`
3. בחר פעולה:
   - **🏷 Tag** — מוסיף תג לכל הנבחרים (תיבת prompt לשם התג)
   - **👤 Assign ▾** — dropdown של fundraisers; לחץ על שם להעביר את כל הנבחרים
   - **🗑 Delete** — שולח לסל המחזור
4. **Clear** מבטל את הבחירה

### ייצוא רשימת תורמים ל-CSV
1. סנן/מיין את הרשימה כרצונך
2. בפינה הימנית למעלה: **⬇ Export CSV**
3. הקובץ יורד עם השם `donors-YYYY-MM-DD.csv`
4. ניתן לפתוח באקסל — UTF-8 BOM מוכלל כך שעברית נראית תקין

### מחיקת תורם
1. סמן את התורם ב-checkbox
2. **Delete 1** הכתום
3. אשר → התורם עובר לסל המחזור
4. **30 יום** לשחזור דרך `Recycle Bin`
5. **שים לב:** מחיקת תורם מוחקת גם את כל הפלאגים, התשלומים, השיחות, וההערות שלו (בקסקייד), אבל הכל ניתן לשחזור

## 4. שיחות, הערות, ומעקבים

### תיעוד שיחה
1. פתח פרופיל תורם
2. לשונית **Calls**
3. **+ Log call**
4. הזן:
   - תאריך + שעה (ברירת מחדל: עכשיו)
   - כיוון: **Outbound** (אתה התקשרת) / **Inbound** (התורם התקשר)
   - ערוץ: Phone / WhatsApp / Email / Meeting
   - משך (אופציונלי)
   - **Outcome**: Answered / Voicemail / No answer / Confirmed donation / Refused / Other
   - **Summary**: סיכום קצר מה דובר
   - **Transcript**: תמלול מלא (אופציונלי)
5. **Save** — השיחה מתועדת ומופיעה בלשונית Calls + ב-Last contact

### הוספת הערה
1. פרופיל תורם → לשונית **Notes**
2. כתוב הערה בתיבת הטקסט
3. **Save**
4. **Pin** — סמן הערה חשובה כך שתופיע ראשונה

### תזמון מעקב (Follow-up)
1. פרופיל תורם → לשונית **Schedule**
2. **+ Schedule follow-up**
3. הזן:
   - **Title** (לדוגמה "להתקשר לגבי הקמפיין השנתי")
   - תאריך + שעה
   - **Kind**: Call / Email / Meeting / Task
   - **Priority**: Low / Normal / High
   - **Remind X minutes before** (אופציונלי)
4. **Save** — יופיע בלוח השנה ובדף Today שלך ביום הרלוונטי

### מעקב יומי — דף Today
- מציג: מעקבים להיום, תשלומים שעברו תאריך, ימי הולדת השבוע, יארצייטים, אימיילים שצריך לשלוח
- **תפקיד**: לפתוח את הדף בבוקר ולעבור על הכל

## 5. פלאגים (הבטחות תרומה)

### מה זה פלאג?
פלאג = **הבטחה** של תורם לתרום סכום מסוים. שונה מתשלום — שזה הכסף שכבר נכנס בפועל. פלאג יכול להיות:
- **חד-פעמי** (Lump sum) — סכום אחד
- **תשלומים** (Installments) — נחלק ל-N תשלומים (חודשי, שבועי, רבעוני, שנתי)
- **Standalone** (synthetic) — תורם תרם בלי פלאג מקדים; המערכת יוצרת אוטומטית פלאג סינתטי כדי שהתשלום ישב על משהו

### הוספת פלאג
1. פתח פרופיל תורם
2. **+ Add pledge**
3. הזן:
   - **Amount** — סכום הפלאג הכולל
   - **Pledge date** — מתי הופלאג נחתם (ברירת מחדל: היום)
   - **Project** — שייך לקמפיין (אופציונלי)
   - **Installments total** — לכמה תשלומים לחלק (1 = lump sum)
   - **Payment plan**: lump_sum / weekly / monthly / quarterly / annual
   - **Payment day** (לחודשי/רבעוני/שנתי) — באיזה יום בחודש
   - **Collection mode**: manual / auto_charge (מצריך כרטיס שמור)
   - הערות
4. **Save** — המערכת יוצרת אוטומטית שורת תשלום (`installment`) לכל מועד

### עריכת פלאג
1. פרופיל תורם → לשונית Giving → לחץ על שורת הפלאג
2. **או** מדף Payments → לחץ על **Pledge** בעמודת הפעולות של שורת תשלום
3. נפתח מודאל עריכה — ערוך סכום, פרויקט, סטטוס, הערות
4. **View payments** — כפתור בתחתית: פותח את דף התשלומים מסונן לפלאג זה בלבד
5. **Save changes**

### מחיקת פלאג
1. במודאל עריכת פלאג, **Delete pledge** (כפתור כתום בפינה השמאלית)
2. אם הפלאג ללא תשלומים — מחיקה ישירה לסל המחזור
3. אם יש תשלומים, בחר מה לעשות:
   - **Delete all** — מוחק גם את התשלומים
   - **Move to another pledge** — מעביר את התשלומים לפלאג אחר של אותו תורם
   - **Convert to standalone donations** — הופך כל תשלום לתרומה חד-פעמית עצמאית
4. הפלאג עובר לסל המחזור (30 יום לשחזור)

### ביקורת פלאגים — מציאת פלאג חריג
**מתי**: אם הסכום ב-"Open pledges" בדף הראשי נראה לא הגיוני.
1. דף ראשי → קישור כחול מתחת לקופסאות: **"Does the Open Pledges total look wrong?"**
2. נפתח מודאל של 20 הפלאגים הפתוחים הגדולים ביותר לפי outstanding
3. פלאגים יתומים (תורם נמחק) יופיעו עם רקע כתום ותג "Orphan"
4. **Delete** לכל פלאג חריג → סל המחזור
5. הדף הראשי מתעדכן מיד

## 6. תשלומים וחיובים

### ביצוע תשלום חדש
1. סרגל עליון → **New payment**
2. **Step 1: בחר תורם**
   - חפש בשם / אימייל / ארגון
   - או הגע דרך `?donor=ID` ב-URL (מהפרופיל)
3. **Step 2: בחר מצב**
   - **Pay against existing pledge** — תשלום על פלאג קיים
   - **New donation** — תרומה חד-פעמית
   - **Split** — חלוקה בין מספר פלאגים/פרויקטים
4. **Step 3: סכום + פרטים**
   - סכום (לכל הקצאה אם split)
   - **Method**: Credit card / Check / Cash / Wire transfer / Other
   - **Project** (אם חדש)
   - **Charge mode**:
     - **Charge now** — חיוב מיידי
     - **Schedule for date** — חיוב בתאריך עתידי (אוטומטי)
     - **Recurring monthly** — חיוב חודשי קבוע
5. **אם Credit card**: יופיעו שדות אשראי מאובטחים (iframes של Sola/Cardknox)
   - תורם מקליד מספר כרטיס בתוך iframe — השרת **לא רואה** את הכרטיס
   - אופציה: **💾 Save card for future charges**
   - אופציה: **📅 Charge monthly automatically** (תשלום חוזר)
   - אופציה: **Don't send receipt email** (לא לשלוח אימייל קבלה)
6. **Charge** — התשלום מתבצע

### תשלום מתוזמן (חיוב בתאריך עתידי)
1. New payment → **Charge mode = Schedule for date**
2. בחר תאריך עתידי
3. הזן פרטי כרטיס + **Save card** (חובה לתזמון)
4. Submit — נוצרת שורה ב-Collections עם status `scheduled`
5. **09:00 UTC** בתאריך הרצוי — הקרון מחייב אוטומטית
6. אם החיוב נכשל, השורה מקבלת status `failed` ועוברת ל-Bounced ב-Collections

### תשלום חודשי אוטומטי
1. New payment → **Charge mode = Recurring monthly**
2. הזן:
   - **Number of charges** — לדוגמה 12 ל-12 חודשים
   - **Day of month** — באיזה יום בחודש לחייב
   - **Charge first installment now** — האם לחייב מיידית את הראשון
3. הזן פרטי כרטיס — נשמר אוטומטית
4. Submit — נוצרות N שורות ב-Collections, כל אחת בתאריך החודשי
5. **09:00 UTC** בכל חודש — הקרון מחייב את הכרטיס הבא
6. **כדי להפסיק**: מחק את השורות שטרם בוצעו דרך Collections

### חיוב מיידי עם כרטיס שמור
1. פרופיל תורם → פאנל **Saved cards**
2. **💳 Charge** ליד כרטיס
3. הזן סכום + פרויקט
4. Confirm → החיוב מתבצע מיד דרך הכרטיס

### ניסיון חיוב מחדש לתשלום שנכשל
1. **Collections** → סנן **Bounced / Failed**
2. **↻ Retry** (כפתור כתום) בשורה
3. נפתח מודאל עם בחירת כרטיס שמור (אם יש) או טופס לכרטיס חדש
4. **Confirm** → ניסיון חיוב חדש

### שליחת קבלה ידנית
1. **Payments** → מצא שורת תשלום (status = paid)
2. **📧 Receipt** בעמודת הפעולות
3. המייל נשלח לכתובת התורם
4. אישור כ-toast למטה

### סינון תשלומים
מעל הטבלה ב-Payments:
- **Status chips**: All / Paid / Scheduled / Bounced / Cancelled / Pending
- **Method chips** (multi-select): Credit card / Check / Cash / Wire / וכו'
- **Type**: All / From a pledge / Standalone donations
- **URL parameter** `?pledge_id=X` — מציג רק תשלומים של פלאג מסוים
- **חיפוש חופשי** + טווח תאריכים

### ביקורת תשלומים
**מתי**: אם ה-"Paid total" נראה מוגזם.
1. דף Payments → קישור כחול מתחת לקופסאות: **"See the 20 largest paid payments"**
2. רשימה של 20 התשלומים הגדולים בסטטוס paid
3. **Edit** על שורה חריגה → פותח את עורך התשלום למחיקה/תיקון

## 7. כרטיסי אשראי שמורים

### שמירת כרטיס
**אוטומטית** כשמסמנים `💾 Save card for future charges` בעת תשלום באשראי. אחרי שהחיוב מצליח, הכרטיס נשמר בכספת המאובטחת של Sola/Cardknox. **השרת שלנו לא רואה את מספר הכרטיס המלא** — רק 4 ספרות אחרונות, חברת הכרטיס, ותאריך תוקף.

### ניהול כרטיסים שמורים
1. פרופיל תורם → פאנל **Saved cards**
2. כל כרטיס מציג:
   - 4 ספרות אחרונות + חברה (Visa / Mastercard / Amex)
   - תאריך תוקף (אדום אם פג)
   - שם בעל הכרטיס
   - תאריך שימוש אחרון
3. **Set as default** — קובע כברירת מחדל לחיובים אוטומטיים
4. **Remove** — מסיר את הכרטיס (לא ניתן לשחזור — אבל הוא נמחק גם מ-Sola)
5. **💳 Charge** — לבצע חיוב מיידי עם כרטיס זה

## 8. קמפיינים (פרויקטים)

### יצירת קמפיין חדש
1. סרגל עליון → **Campaigns** → **+ New campaign**
2. הזן:
   - **Name** (חובה) — לדוגמה "Annual Drive 2026"
   - **Description**
   - **Goal amount** — יעד גיוס (אופציונלי)
   - **Start date** / **End date**
   - **Parent campaign** — אם זה תת-קמפיין של אחר
3. **Save**

### מעקב התקדמות קמפיין
- כל כרטיס בדף Campaigns מציג:
  - **Raised** (סכום שגויס) — כולל פלאגים פתוחים, לא רק כסף שנכנס
  - **Goal** (יעד)
  - **Progress bar** — ירוק עד 100%, **זהב מעל 100%** עם תווית 🎯 "Goal achieved"
  - **מספר תורמים** ייחודיים
  - **סכום פלאג כולל**

### Prospects (מועמדים) לקמפיין
**מה זה**: רשימת call-list פרטית של תורמים שאתה חושב שיתרמו לקמפיין. **לא** נספר בסכום הפלאגים הרשמי.

1. פרופיל קמפיין → פאנל **Campaign prospects**
2. **+ Add prospect**
3. בחר תורם + הזן:
   - **Estimated amount** — כמה אתה מעריך שיתרום
   - **Status**: pending / called / confirmed / declined
   - **Notes**
4. אחרי שתורם אכן תורם → מחק את המועמד או סמן כ-confirmed

### שליחת דיוור לקמפיין
1. פרופיל קמפיין → **📧 Email campaign donors**
2. **Recipients** (קבוצה לשליחה):
   - **Campaign donors** — תורמי הקמפיין (יש להם פלאג בקמפיין)
   - **Prospect list** — רק המועמדים
   - **Open pledgers** — תורמים עם פלאג פתוח (לא שולם במלואו)
   - **All donors** — כל התורמים הפעילים בארגון
3. **Start from a saved template** (אופציונלי) — טוען subject + body מתבנית קיימת
4. **Subject** + **Body** (HTML מותר)
5. השתמש במשתנים:
   - `{{first_name}}` / `{{last_name}}` / `{{full_name}}` / `{{hebrew_name}}`
6. **🔍 Test send** — שלח לעצמך בדיקה לפני שליחה אמיתית
7. **Send blast** — שליחה לכל הקבוצה (10 בשנייה)
8. תקבל סיכום: X sent, Y failed, רשימת שגיאות

**הגנה אוטומטית**: תורמים עם `email_opt_in='none'` או `do_not_contact=1` לא יקבלו דיוורים, גם אם הם בקבוצה.

## 9. אימייל ותבניות

### מערכת התבניות
**4 סוגי תבניות**:
1. **Receipt** — אימייל קבלה אוטומטי אחרי חיוב (לפחות אחת, מסומנת כ-default)
2. **Campaign blast** — תבניות לדיוורים
3. **Thank you** — תודות ידניות לתורמים גדולים
4. **Custom** — כל אימייל אחר

### יצירת תבנית חדשה
1. תפריט משתמש → **Emails**
2. בקטגוריה הרצויה, **+ (פלוס)** ליד שם הקטגוריה
3. הזן:
   - **Name** — פנימי, רק לך
   - **Subject** — נושא האימייל (תומך במשתנים)
   - **Body (HTML)** — גוף האימייל
4. **כפתורי משתנים** מתחת לתיבת הגוף:
   - `{{first_name}}` / `{{hebrew_name}}` / `{{amount}}` / `{{paid_date}}` / `{{method}}` / `{{project_name}}` / `{{transaction_ref}}` / `{{receipt_number}}` / `{{organization_name}}` ועוד
   - לחיצה — מוסיף את המשתנה במיקום הסמן
5. **לשונית Preview** — רואים איך נראה האימייל עם נתוני דוגמה
6. **Save / Create**

### הגדרת תבנית קבלה כברירת מחדל
1. פתח תבנית מסוג Receipt
2. סמן **Set as default receipt** (קופסה ירוקה)
3. Save
4. **מעכשיו**: כל קבלה אוטומטית אחרי חיוב — תשתמש בתבנית הזו
5. אם אין תבנית default — נשמר ה-HTML הברירת מחדל המוטמע

### עריכת תבנית קיימת
1. עמודה שמאלית: לחץ על שם תבנית
2. עורך פותח: ערוך Name / Subject / Body
3. **Save** (כפתור ימני)
4. **Preview** לפני שמירה

### הגדרות אימייל כלליות
`Settings → Email`:
- **From** — כתובת השולח (חייב להיות verified ב-Resend)
- **Signature** — HTML שיתווסף לסוף כל אימייל
- **Resend API key** — נשמר מוצפן

## 10. גבייה (Collections)

### מה זה
דף שמרכז **חיובים שצריך לטפל בהם** — שעברו תאריך, חוזרו (bounced), או נכשלו.

### Tabs
- **Overdue** (ברירת מחדל) — חיובים שעברו תאריך
- **Bounced / failed** — צ'קים חזרו / חיובי אשראי נכשלו
- **Upcoming** — צפויים ב-7 ימים הקרובים
- **All open** — הכל

### סינון
- **Donor** — שם / טלפון
- **Project** — dropdown
- **Method** — dropdown
- **Min $ / Max $** — טווח סכום

### פעולות על שורה
- **💳 Charge** / **↻ Retry** — חיוב כעת (כחול לרגיל, כתום לניסיון מחדש)
- **✓ Paid** — סמן כשולם
- **Reschedule** — שנה תאריך
- **Bounce** — סמן כצ'ק חוזר (לצ'קים בלבד)
- **Cancel** — בטל את התשלום

### ייצוא והדפסה
- **⬇ Export CSV** — קובץ CSV של השורות הנוכחיות
- **🖨 Print / PDF** — חלון הדפסה (אפשר לשמור כ-PDF)
- ההדפסה כוללת כותרת אוטומטית: סוג הלשונית, תאריך יצירה, הסינון הפעיל

## 12. סל מחזור

### מה זה
**שחזור מחיקות עד 30 יום אחורה** — לכל תורם, פלאג, או תשלום שנמחק.

### צפייה בסל המחזור
1. תפריט משתמש → **Recycle Bin**
2. רשימה: סוג / סיכום / נמחק על-ידי / מתי / **Auto-purge in X days**
3. **Chips לסינון**: All / Donors / Pledges / Payments

### שחזור פריט
1. מצא את הפריט ברשימה
2. **↺ Restore** (כפתור כהה)
3. הפריט חוזר למצב המקורי
4. **תורם** — כל הפלאגים, תשלומים, שיחות, הערות שלו חוזרים גם
5. **פלאג** — הוא ותשלומיו חוזרים
6. **תשלום** — רק הוא חוזר

### מחיקה לצמיתות (מיידית)
1. **Delete forever** (כפתור כתום)
2. אישור — הפריט נמחק מהמערכת לחלוטין
3. **אין שחזור** אחרי זה

### ניקוי אוטומטי
**כל יום ב-04:00 UTC**, הקרון מוחק לצמיתות את כל מה שמעל 30 יום בסל המחזור.

## 17. שאלות נפוצות (FAQ)

### למה תורם לא מקבל קבלה?
1. בדוק שיש לתורם **אימייל** בפרופיל
2. בדוק שלא סומן **Don't send receipt email** בעת התשלום
3. בדוק ש-Resend מחובר ב-Settings → Email
4. בדוק את **Audit Log** או את **fr_email_log** ב-DB לראות אם נשלח
5. בדוק את ה-spam folder של התורם
6. שלוח ידנית: Payments → **📧 Receipt** בשורת התשלום

### למה ה-"Open pledges" בדף הראשי מציג סכום מוגזם?
- הקופסה מציגה את כל הפלאגים הפתוחים (status='open') במערכת
- ייתכן יש פלאג בדיקה ישן או פלאג עם סכום שגוי
- לחץ על **"See the 20 largest open pledges"** ובחר מה למחוק
- פלאגים יתומים (תורם נמחק) יסומנו

### למה תורם לא מופיע ברשימה?
- בדוק את הסטטוס שלו: Lead vs Donor
- אם אתה fundraiser: התורם חייב להיות מוקצה אליך
- בדוק את החיפוש / סינון — אולי יש Lapsed/Open-pledges פעיל
- אולי הוא בסל המחזור (Recycle Bin)

### האם אפשר לבטל חיוב שכבר התבצע?
- במערכת: סמן את התשלום כ-`cancelled` או מחק אותו
- בכרטיס: לבצע refund ב-Cardknox dashboard (לא נתמך מכאן)
- מומלץ: ליצור תשלום חדש עם סכום שלילי כתיעוד החזר

### איך מייצאים נתונים לרואה חשבון?
- **דרך 1**: Reports → סנן לפי שנה → **⬇ Export CSV**
- **דרך 2**: Reports → **🖨 Print / PDF** → Save as PDF
- **דרך 3**: Payments → סנן → Export CSV

### מה זה Standalone donation?
- תרומה חד-פעמית שלא הייתה לפניה פלאג (התורם פשוט תרם)
- המערכת יוצרת באופן אוטומטי פלאג סינתטי (is_standalone=1) כדי שהתשלום ישב על משהו
- לא נספר בסכום הפלאגים — רק בסכום ששולם

### איך לסגור קמפיין שהסתיים?
1. Campaigns → לחץ על הקמפיין
2. **Archive** — הוא יעבור ל-"Archived & closed"
3. ההיסטוריה נשמרת
4. ב-Reports עדיין ניתן לראות אותו עם סינון

### האם המידע מאובטח?
- **כרטיסי אשראי**: מספרים מלאים לעולם **לא** עוברים דרך השרת שלנו. Sola/Cardknox מטפלים בטוקניזציה ב-iframes נפרדים.
- **סיסמאות**: hashed bcrypt בשרת
- **API keys**: מוצפנים ב-DB
- **HTTPS**: בכל בקשה (Vercel)
