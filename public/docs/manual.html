1. מבוא והתחלת עבודה

מה המערכת עושה

המערכת היא CRM לעמותות גיוס. היא מאפשרת:

  • ניהול בסיס נתונים של תורמים ולידים
  • רישום הבטחות תרומה (פלאגים) וחיובים
  • חיוב אוטומטי בכרטיס אשראי דרך Sola/Cardknox
  • שליחת קבלות אוטומטיות ודיוורים
  • מעקב אחר גבייה (חיובים שעברו תאריך)
  • דוחות ותקציבים
  • שיתוף פעולה צוותי (מנהל + fundraisers)

תהליך התחלת עבודה (Setup ראשוני)

למנהל בלבד, פעם אחת:

  1. חיבור שער התשלומים — תפריט משתמש → Settings → Sola/Cardknox — הזן PaymentSITE URL, xKey, ifields key
  2. חיבור שליחת אימיילים — Settings → Email — From address, Resend API key, signature, ושלח test email
  3. אימות דומיין ב-Resend — הוסף 3 רשומות DNS (SPF, DKIM, MX)
  4. הוספת תורם ראשון — Donors → New donor
  5. יצירת קמפיין ראשון — Campaigns → + New campaign

2. מבנה הממשק

הסרגל העליון (Navigation)

  • Dashboard — דף ראשי עם סיכומים
  • Today — מבט יומי
  • Leads — אנשים שעדיין לא תרמו
  • Donors — תורמים פעילים
  • Campaigns — מיזמי גיוס
  • New payment — ביצוע תשלום
  • Payments — היסטוריית תשלומים
  • Collections — חיובים שצריך לטפל
  • Calendar — לוח שנה
  • Reports — דוחות
  • Team / Import — מנהל בלבד

תפריט שם המשתמש

לחיצה על שמך בפינה הימנית-עליונה → Emails, Audit Log, Recycle Bin, Settings, Sign out (כולם מנהל בלבד למעט Sign out).

חיפוש מהיר

⌘K (Mac) או Ctrl+K (Windows) — חיפוש מכל מקום במערכת.

3. תורמים ולידים

ההבדל

  • Lead — עדיין לא תרם
  • Donor — תרם לפחות פעם אחת (אוטומטית עובר מ-Lead ל-Donor אחרי התרומה הראשונה)

הוספת תורם

  1. Donors / Leads → + New donor / + New lead
  2. מלא: First name (חובה), Last name, Hebrew name, אימייל, טלפון, ארגון, מקור, תאריכים
  3. סמן Do not contact אם רלוונטי
  4. שמור

חיפוש וסינון

תיבת חיפוש (שם/אימייל/ארגון), סינון לפי Source, chips "Lapsed (12m+)" ו-"Has open pledges", מיון לכל עמודה בלחיצה (3 מצבי גלגול: none → desc → asc).

פעולות גורפות

  1. סמן 2+ תורמים
  2. סרגל כחול נפתח
  3. בחר: 🏷 Tag / 👤 Assign / Delete

ייצוא ל-CSV

סנן ומיין → ⬇ Export CSV → קובץ עם UTF-8 BOM (אקסל יציג עברית תקין).

4. שיחות, הערות, ומעקבים

תיעוד שיחה

פרופיל תורם → Calls → + Log call → תאריך, כיוון (in/out), ערוץ, outcome, summary, transcript אופציונלי.

הוספת הערה

פרופיל תורם → Notes → כתוב + Save. הערות חשובות → Pin.

תזמון Follow-up

פרופיל תורם → Schedule → + Schedule follow-up → title, תאריך/שעה, kind, priority, תזכורת. יופיע בלוח השנה ובדף Today.

5. פלאגים (הבטחות תרומה)

מה זה פלאג?

הבטחה של תורם לתרום. שונה מתשלום (שזה הכסף שכבר נכנס).

  • חד-פעמי (Lump sum)
  • תשלומים (Installments: weekly / monthly / quarterly / annual)
  • Standalone — תורם תרם בלי פלאג מקדים; המערכת יוצרת פלאג סינתטי

הוספת פלאג

  1. פרופיל תורם → + Add pledge
  2. Amount, Pledge date, Project, Installments total, Payment plan, Payment day, Collection mode, הערות
  3. Save — המערכת יוצרת installment row לכל מועד

עריכת/מחיקת פלאג

פרופיל תורם → Giving → לחץ על שורת פלאג. במודאל: View payments פותח דף תשלומים עם סינון לפלאג.

למחיקה: Delete pledge → אם יש תשלומים בחר: Delete all / Move to another pledge / Convert to standalone donations.

ביקורת פלאגים (audit)

דף ראשי → קישור "Does the Open Pledges total look wrong?" → מודאל 20 הפלאגים הגדולים, עם זיהוי יתומים (orphans).

6. תשלומים וחיובים

ביצוע תשלום

  1. New payment → בחר תורם → בחר mode (existing pledge / new donation / split)
  2. סכום, method, project
  3. Charge mode: Charge now / Schedule for date / Recurring monthly
  4. אם credit card: iframes של Sola, אופציה Save card, אופציה Skip receipt
  5. Charge

תשלום מתוזמן

Charge mode = Schedule for date → תאריך → Save card (חובה) → Submit. ב-09:00 UTC ביום הרצוי הקרון מחייב.

תשלום חודשי אוטומטי

Charge mode = Recurring monthly → מספר חיובים + יום בחודש + אופציה לחיוב מיידי + Save card. הקרון יחייב כל חודש.

ניסיון חיוב מחדש

Collections → Bounced / Failed → ↻ Retry → מודאל עם בחירת כרטיס שמור או טופס חדש.

שליחת קבלה ידנית

Payments → 📧 Receipt בעמודת הפעולות (רק לתשלומים paid).

סינון תשלומים

Status chips + Method chips (multi) + Type filter + ?pledge_id=X URL param.

ביקורת תשלומים

Payments → קישור "See the 20 largest paid payments" → רשימה למחיקה / תיקון של חריגים.

7. כרטיסי אשראי שמורים

שמירה אוטומטית כשמסמנים 💾 Save card בעת חיוב. השרת לא רואה את המספר המלא, רק 4 ספרות אחרונות + תאריך תוקף.

ניהול: פרופיל תורם → פאנל Saved cards → Set as default / Remove / 💳 Charge.

8. קמפיינים (פרויקטים)

יצירה

Campaigns → + New campaign → Name (חובה), Description, Goal amount, תאריכים, Parent campaign (לתת-קמפיינים).

מעקב התקדמות

כל כרטיס מציג: Raised / Goal / Progress bar (ירוק עד 100%, זהב מעל, תווית 🎯 "Goal achieved"). מספר תורמים, סכום פלאג.

Prospects

רשימת call-list פרטית. פרופיל קמפיין → Campaign prospects → + Add prospect → תורם, estimated_amount, status. לא נספר בסכום הרשמי.

דיוור לקמפיין

  1. פרופיל קמפיין → 📧 Email campaign donors
  2. Recipients: Campaign donors / Prospect list / Open pledgers / All donors
  3. Start from saved template (אופציונלי)
  4. Subject + Body (HTML + משתנים)
  5. 🔍 Test send — חובה לבדוק לפני שליחה!
  6. Send blast → 10 בשנייה, סיכום בסוף
תורמים עם do_not_contact=1 או email_opt_in='none' לא יקבלו דיוורים אוטומטית.

9. אימייל ותבניות

4 סוגי תבניות

  • Receipt — אוטומטית אחרי חיוב (default 1)
  • Campaign blast — לדיוורים
  • Thank you — תודות ידניות
  • Custom — אחר

יצירה

תפריט משתמש → Emails → + ליד קטגוריה → Name, Subject, Body. כפתורי משתנים: {{first_name}}, {{hebrew_name}}, {{amount}}, {{paid_date}}, {{project_name}}, {{transaction_ref}}, {{receipt_number}}, {{organization_name}}.

תבנית קבלה ברירת מחדל

תבנית מסוג Receipt → סמן Set as default receipt → Save. מעכשיו כל קבלה אוטומטית משתמשת בה.

10. גבייה (Collections)

Tabs

Overdue / Bounced+failed / Upcoming (7d) / All open.

סינון

Donor name/phone, Project, Method, טווח סכום.

פעולות שורה

  • 💳 Charge / ↻ Retry — חיוב כעת
  • ✓ Paid — סמן כשולם
  • Reschedule — שנה תאריך
  • Bounce — סמן צ'ק חוזר
  • Cancel — בטל

ייצוא והדפסה

⬇ Export CSV + 🖨 Print / PDF בראש הדף. כותרת הדפסה אוטומטית עם הקשר הסינון.

11. דוחות

KPIs

Total raised, Donors, Payments, Avg payment, Outstanding pledged, Open pledge count.

פילטרים (Excel-style)

From/To, Projects, Sources, Donors (עם חיפוש), Fundraisers (מנהל). dropdowns עם checkboxes + Select all / Clear. presets: Last 30/90/YTD/All time.

Charts

By month (bar chart), By project, By source, By method.

Top donors / Lapsed donors

12 הגדולים בתקופה / 25 שלא תרמו 12 חודש (ממוין lifetime value).

פאנל הפלאגים (Pledges)

הכי שימושי: כל פלאג עם Pledged / Paid / Remaining + Progress bar + Status badge + Payments link. Status chips (בעברית), חיפוש, Totals מתעדכן, Export CSV, 🖨 Print / PDF משלו (מדפיס רק את הפאנל הזה).

ייצוא דוח כולל

⬇ Export CSV (ירוק) + 🖨 Print / PDF (שחור) בראש הדף.

12. סל מחזור

תפריט משתמש → Recycle Bin. שחזור עד 30 יום של תורם / פלאג / תשלום. אחרי 30 יום — קרון יומי ב-04:00 UTC מנקה לצמיתות.

↺ Restore — מחזיר למצב המקורי (כולל כל הdependencies — תורם שמשוחזר מקבל בחזרה את כל הפלאגים והתשלומים).

Delete forever — מחיקה מיידית לצמיתות (אין שחזור).

13. יומן ביקורת (Audit Log)

תפריט משתמש → Audit Log (מנהל בלבד). רישום כל שינוי: מתי, מי, מה, סיכום, סוג entity. Chips לסינון + תיבת חיפוש. כל מחיקה/עדכון של תורם/פלאג/תשלום, וכל שליחת דיוור — נרשמים.

14. ניהול צוות

מנהל בלבד. Team → + New fundraiser → שם + אימייל → סיסמה זמנית נשלחת. ה-fundraiser רואה רק את התורמים המוקצים אליו. הקצאה: בודד דרך פרופיל / מאסיבי דרך Donors → 👤 Assign.

15. הגדרות מערכת

  • Payment gateway — Sola/Cardknox PaymentSITE URL, xKey, ifields key
  • Email — From, Signature, Resend key, Send test
  • Sola Sync — סנכרון חיובים שבוצעו ישירות ב-Cardknox
  • Auto-reminder cron — endpoint לקרון תזכורות
  • 📖 מדריך שימוש — האינדקס הזה

16. קיצורי מקלדת וטיפים

  • ⌘K / Ctrl+K — Command Palette

  • /fundraising/payment?donor=ID — תשלום ישיר לתורם

  • מיון: לחיצה על כותרת (cycling: none → desc → asc → none)

17. שאלות נפוצות (FAQ)

למה תורם לא מקבל קבלה?

  1. בדוק שיש לתורם אימייל בפרופיל
  2. בדוק שלא סומן Don't send receipt email
  3. בדוק ש-Resend מחובר (Settings → Email)
  4. בדוק Audit Log
  5. בדוק spam folder
  6. שלוח ידנית: Payments → 📧 Receipt

למה ה-Open pledges מוגזם?

דף ראשי → "See the 20 largest open pledges" → מצא ומחק חריג.

למה תורם לא מופיע?

בדוק status (Lead/Donor), הקצאה אם אתה fundraiser, סינונים פעילים, סל המחזור.

איך לבטל חיוב שכבר התבצע?

במערכת: סמן cancelled. בכרטיס: refund ב-Cardknox dashboard. מומלץ: ליצור תשלום שלילי כתיעוד החזר.

איך מייצאים לרואה חשבון?

Reports → Export CSV / Print PDF.

מה זה Standalone donation?

תרומה חד-פעמית ללא פלאג מקדים. המערכת יוצרת פלאג סינתטי (is_standalone=1) כדי שהתשלום ישב על משהו.

איך לסגור קמפיין?

Campaigns → לחץ על קמפיין → Archive. ההיסטוריה נשמרת.

האם המידע מאובטח?

כרטיסי אשראי: רק טוקנים, השרת לא רואה מספרים מלאים. סיסמאות: bcrypt. API keys: מוצפנים. HTTPS תמיד. גיבויים יומיומיים של Turso.
